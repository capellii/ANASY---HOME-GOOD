# Guia de Contribuição ANASY

## Padrões de Código
- Use TypeScript no backend
- Siga Clean Architecture e SOLID
- Use ESLint e Prettier para garantir qualidade
- Nomeie arquivos e pastas em inglês, com nomes claros

## Commits
- Use mensagens curtas e objetivas
- Exemplo: `feat: add device controller`
- Commits de documentação: `docs: update API reference`

## Pull Requests
- Sempre descreva o que foi feito
- Adicione screenshots se for UI
- Teste antes de pedir revisão

## Testes
- Escreva testes unitários para cada serviço e controller
- Use Jest no backend

## Revisão
- Todo código será revisado por pelo menos 1 dev
- Não faça merge direto na main sem aprovação

## Convenções
- Variáveis: camelCase
- Classes: PascalCase
- Funções: camelCase
- Constantes: UPPER_SNAKE_CASE
